# Pair Trading Project - Setup and Run Guide

This README only covers what you need to install dependencies and run the project.

## 1. Prerequisites

1. Python 3.10 or newer
2. VS Code
3. VS Code extensions:
1. Python
2. Jupyter

## 2. Install Required Libraries

Open PowerShell in the project root and run:

python -m venv .venv

.\.venv\Scripts\Activate.ps1

python -m pip install --upgrade pip

python -m pip install yfinance pandas numpy matplotlib seaborn statsmodels scikit-learn scipy joblib tqdm streamlit plotly

## 3. Run Notebooks in Order

Run these notebooks top to bottom, in this exact sequence:

1. [notebooks/01_data_collection.ipynb](notebooks/01_data_collection.ipynb)
2. [notebooks/02_eda_pairs.ipynb](notebooks/02_eda_pairs.ipynb)
3. [notebooks/03_pair_selection.ipynb](notebooks/03_pair_selection.ipynb)
4. [notebooks/04_signal_backtest.ipynb](notebooks/04_signal_backtest.ipynb)
5. [notebooks/05_ml_filter_random_forest.ipynb](notebooks/05_ml_filter_random_forest.ipynb)

If you are using the alternate Phase 3 notebook, run [notebooks/03_pair_selection1.ipynb](notebooks/03_pair_selection1.ipynb) instead of notebook 3 above.

Important:

1. Use the same Python environment (.venv) for all notebooks.
2. Do not skip order, because each notebook depends on files generated by previous notebooks.

## 4. Run the Dashboard

From project root:

.\.venv\Scripts\Activate.ps1

python -m streamlit run dashboard.py --server.port 8501

Open in browser:

http://localhost:8501

After notebooks complete, these files should exist:

1. [data/nifty_prices_clean.csv](data/nifty_prices_clean.csv)
2. [data/selected_pairs.csv](data/selected_pairs.csv)
3. [data/baseline_backtest_stats.csv](data/baseline_backtest_stats.csv)
4. [data/ml_vs_baseline_stats.csv](data/ml_vs_baseline_stats.csv)
5. [outputs/06_ml_equity_and_importance.png](outputs/06_ml_equity_and_importance.png)

## 6. Common Fixes

1. If a library is missing:
python -m pip install package_name
2. If a ticker fails in yfinance, update the ticker list in Notebook 1 and rerun from there.
3. If downstream files are missing, rerun notebooks from Phase 1 in order.
